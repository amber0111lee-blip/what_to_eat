# 觅食 · 大学城美食地图

> 一款面向北京高校联盟大学生的本地化吃饭决策工具

![版本](https://img.shields.io/badge/版本-v4.0-orange) ![平台](https://img.shields.io/badge/平台-移动端优先-blue) ![地图](https://img.shields.io/badge/地图-高德API-green) ![技术栈](https://img.shields.io/badge/技术栈-原生HTML%2FCSS%2FJS-lightgrey)

## 项目简介

**觅食**不是普通的餐厅信息聚合页，而是帮助大学生**更快做出"今天吃什么"决策**的产品 Demo。

核心痛点来自真实的大学生就餐场景：信息分散在外卖平台、点评平台、学校公众号中，学生面对「食堂 / 外卖 / 校外堂食」三类选择时决策成本极高，且现有平台缺少学生视角标签（预算、健康、距离、进校成本、社交场景）。

本项目覆盖北京高校联盟：**中国人民大学、中国农业大学、北京大学、清华大学**周边餐饮场景，共收录 15 条示例餐饮数据（含 5 条外卖专属）。

## 快速开始

直接下载 `university-food-map.html`，用浏览器打开即可运行，**无需任何服务器或依赖安装**。

```bash
git clone https://github.com/你的用户名/university-food-map.git
cd university-food-map
open university-food-map.html   # macOS
# 或直接双击文件用浏览器打开
```

> 地图功能需要联网加载高德地图 API，其他功能完全离线可用。

## 功能模块

### 🏠 首页 — 场景决策优先

首屏以「帮你做决定」为第一入口，而非展示推荐流。

**场景入口宫格（6 个）**

用情境式问题替代传统分类标签，直接触发决策：

| 入口 | 作用 |
|------|------|
| 🧘 一个人吃什么 | 过滤一人食友好商家 |
| 👫 和朋友聚餐吃什么 | 过滤聚餐场景及食堂 |
| 🪙 20元以内吃什么 | 过滤预算 20 元以内选项 |
| 🥗 想吃得健康一点 | 过滤蔬菜友好 / 轻食商家 |
| ⚡ 不想等太久 | 过滤出餐快的商家 |
| 🍱 食堂还是点外卖 | 直接打开 AI 决策向导 |

**高峰期预警横幅**

根据当前时段实时判断整体拥挤程度，分三级（⚠️ 高峰期 / ✅ 正常 / 🌿 空闲），给出错峰建议。

**发现页入口卡片**

三张快捷卡片（20元内推荐 / 宿舍外卖保底 / 食堂窗口），引导进入发现页浏览。

**推荐卡片流**

「附近推荐」和「最近大家都在吃」两组横向滑动卡片，每张卡片右上角显示实时拥挤状态（排队中 / 稍有人 / 很空闲）。

---

### 🗺️ 地图页 — 高校联盟美食地图

顶部「🎓 高校联盟美食地图」标识栏，明确区别于普通区域餐饮地图。

**三类点位，一眼区分**

| 样式 | 类型 |
|------|------|
| 🔴 橙色圆形 | 校外堂食 |
| 🟢 绿色圆形 | 校内食堂 |
| 🔵 蓝色方形 | 外卖 |

拥挤度越高，点位描边颜色越深（白色 → 黄色 → 红色）。

**六维筛选面板**（点击 ⚙ 打开）

| 维度 | 选项 |
|------|------|
| 用餐方式 | 校外堂食 / 校内食堂 / 外卖点单 |
| 就餐场景 | 一人食 / 多人聚餐 / 夜宵 / 早餐 |
| 人均预算 | 15元内 / 15-30元 / 30-60元 / 60元以上 |
| 健康偏好 ✨ | 清淡 / 蔬菜友好 / 轻食沙拉 / 非油炸 |
| 特殊条件 ✨ | 可跨校进入 / 无需预约 / 出餐快 / 深夜供应 |

筛选后顶部显示已选条件 Chip，可单独点 × 删除；零结果时弹出引导卡。点击地图标记弹出信息卡，含按时段分布的今日拥挤度色条。

---

### ✦ 发现页 — 榜单 + 专题浏览

补充「随便逛逛找灵感」的使用场景，增强产品可逛性。

**四张榜单卡片**
- 💰 20元内也能吃得不错
- 🛵 这几家外卖最适合宿舍保底
- 🏫 最近最值得去的食堂窗口
- 🎉 和朋友聚餐不容易踩雷的店

**应季专题**
- 🌱 春天吃点轻盈的（清爽、蔬菜友好、吃完没负担）
- 🌙 晚八点后还能吃什么（适合晚课后、社团结束后）

**更多主题**

一个人也能安心吃 / 想吃蔬菜的时候 / 适合临时聚一下 / 跨校打卡食堂 / 值得专门出校门吃的店

点击任意榜单或专题，下方直接渲染过滤后的推荐卡片。

---

### 🏫 食堂页 — 高校食堂专区

将校内食堂从「被忽视的信息」变成重要决策选项。

**学校维度 Tab 切换**

全部 / 中国农业大学 / 中国人民大学 / 北京大学 / 清华大学

**每所食堂卡片**

营业状态 + 三道招牌菜 + 实时价格 + 今日拥挤度色条 + 进校提示 + 「查看详情」入口

**跨校打卡攻略（独立模块）**

三所学校各一张攻略卡，回答五个真实决策问题：

| 字段 | 内容示例 |
|------|----------|
| 能不能进 | 刷身份证直接进 / 公众号预约 / 暂停开放 |
| 时间成本 | 骑行距离 + 往返耗时 |
| 额外成本 | 校外学生是否有价格差异 |
| 适合谁去 | 打卡型 / 顺路型 / 不建议专程 |
| 注意事项 | 高峰时段、预约渠道 |

颜色区分推荐程度：🟢 强烈推荐专程去 / 🟡 需提前规划 / 🟠 顺路可考虑

---

### 🤍 详情页 — 决策优先的店铺介绍

从任意卡片或地图点位点击进入，从底部整屏上滑。

**普通店铺结构**

用餐方式标签 → 一句推荐理由（橙色左边框） → 快速信息（价格/距离/评分） → 招牌菜 → 同学短评 3 条 → 今日拥挤度色条 → 特征标签 → 适合人群 → 相似推荐

**外卖详情差异化**

拥挤度替换为「🛵 外卖须知」（预计配送时间、提前下单建议、取餐方式）。

**食堂详情差异化**

额外展示「🚪 去之前先确认这些」（进校方式 / 价格说明 / 错峰建议）。

---

### 🍽️ AI 决策向导 — 5步找到最合适的那顿

点击首页「今天吃什么」打开，5个问题约30秒完成。

| # | 问题 | 决策作用 |
|---|------|---------|
| Q1 | 几个人吃？ | 一人食 / 小聚 / 聚餐场景分流 |
| Q2 | 预算多少？ | ≤15元 / 15-30 / 30-60 / 随便 |
| Q3 | 校内解决还是出校？ | 校内食堂 / 出校堂食 / 点外卖 / 无所谓 |
| Q4 | 快还是好？ | 快点吃上 / 吃得更好 / 性价比 / 都行 |
| Q5 | 今天的状态？（情绪题） | 清淡 / 重口 / 安慰自己 / 别踩雷就行 |

**结果页**：情绪感知总结文案 + 3张推荐卡片 + 「换一批」（每次排除已显示结果）+ 「地图看位置」

---

### 🍜 饭搭子 — 轻量约饭社区

解决「i人不友好」和「周末凑不齐人」的社交场景。

**约饭广场**：浏览他人发布的约饭邀请，按「今天 / 本周末 / 步行可达 / 食堂探店」筛选，显示名额可视化和参与按钮。

**发布约饭**：时间 / 地点 / 口味 / 人数均支持快速选项或「✏️ 自定义」输入。

## 技术说明

| 项目 | 说明 |
|------|------|
| 技术栈 | 纯 HTML / CSS / JavaScript，零框架依赖 |
| 地图 | 高德地图 JS API v2.0（真实坐标） |
| 字体 | Noto Serif SC + Noto Sans SC（Google Fonts CDN） |
| 数据 | 本地示例数据（15条餐厅 + 6所食堂），无后端 |
| 兼容性 | 移动端优先，最大宽度 430px，桌面也可正常访问 |
| 部署 | 单文件，直接双击或托管到任意静态服务 |

### 替换高德地图 Key

文件顶部：

```html
<script>
  window._AMapSecurityConfig = { securityJsCode: '你的安全密钥' };
</script>
<script src="https://webapi.amap.com/maps?v=2.0&key=你的APIKey"></script>
```

前往 [高德开放平台](https://lbs.amap.com/) 申请 Web 端 Key，本地调试时白名单留空即可。

## 数据结构

### 餐厅字段说明

```js
{
  id: 1,
  name: '黄焖鸡米饭',
  emoji: '🍗',
  type: 'cn',              // cn / jpkr / west / snack / canteen
  mealType: 'dine-in',     // dine-in（堂食）/ canteen（食堂）/ delivery（外卖）
  solo: true,              // 是否一人食友好
  price: 20,               // 人均价格，用于筛选
  label: '¥20',            // 展示用
  dist: '步行5min',
  rating: 4.6,
  lat: 39.9969, lng: 116.3316,
  bg: '#FFF3E0',           // 卡片背景色
  tags: ['一人食','快餐','中餐'],
  health: [],              // 健康标签：light / veggie / salad / nonfried
  special: ['fast','noReserve'], // 特殊条件：fast / noReserve / canEnter / latenight
  mood: ['comfort','safe'],     // 情绪匹配（向导第5题）：light / heavy / comfort / safe
  bp: [0,0,1,1,...],       // 24小时拥挤度，0-4级
  oneLiner: '一句推荐理由',
  suitable: ['适合人群标签'],
  dishes: [{ n:'菜名', p:'价格', e:'emoji' }],
  reviews: ['学生短评1', '学生短评2', '学生短评3'],
  entryInfo: {             // 仅食堂类使用
    access: '进校方式',
    price:  '价格说明',
    tips:   '注意事项'
  }
}
```

### 拥挤度等级

`bp` 数组共 24 个元素，下标对应 0-23 时：

| 级别 | 含义 | 色条颜色 |
|------|------|---------|
| 0 | 暂不营业 | 灰色 |
| 1 | 很空闲，随时去 | 绿色 |
| 2 | 稍有人，正常候餐 | 黄色 |
| 3 | 较忙，约等 10-15 分钟 | 橙色 |
| 4 | 高峰期，约等 20 分钟+ | 红色 |

地图标记描边颜色随当前时段的拥挤度变化；首页横幅取所有餐厅当前时段的均值。

## 产品设计原则

- **决策优先**：先帮用户缩小选择范围，再展示详细信息
- **学生友好**：预算、距离、进校成本、健康偏好都是筛选维度
- **信息直观**：标签清楚、价格前置、场景明确
- **氛围轻松**：像一本精致的本地生活指南，而不是外卖 App
